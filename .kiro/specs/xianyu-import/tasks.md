# 任务清单: 闲鱼商品搬运

## 1. Foundation：环境与领域模型准备

- [x] 1.1 引入服务端 HTML 解析依赖并验证环境就绪
  - 安装 cheerio 到项目依赖
  - 在本地开发环境执行解析器初始化，确认无运行时错误
  - Observable：项目可正常启动， cheerio 能被正确 import
  - _Boundary: Scraping Infrastructure_
  - _Requirements: 2.4, 2.5_

- [x] 1.2 定义抓取结果与错误类型的领域模型
  - 定义抓取成功的商品数据结构（标题、价格、描述、图片列表、成色）
  - 定义抓取失败的标准错误码（超时、获取失败、解析失败、非商品页）
  - Observable：类型定义文件存在，且能被领域服务正确引用
  - _Boundary: Scraping Infrastructure_
  - _Requirements: 2.5, 2.7, 2.8, 2.9_

## 2. Core：抓取引擎实现

- [x] 2.1 (P) 实现闲鱼链接格式校验与标准化能力
  - 仅接受标准闲鱼商品详情页域名，拒绝淘口令、短链及其他平台链接
  - 对合法链接做 URL 标准化处理（统一协议与域名格式）
  - Observable：给定合法闲鱼链接返回标准化结果；给定非法链接返回格式无效错误
  - _Boundary: LinkValidator_
  - _Requirements: 1.1, 1.2, 5.1_

- [x] 2.2 (P) 实现闲鱼页面 HTTP 抓取与超时控制
  - 使用匿名 HTTP 请求获取闲鱼商品页面 HTML
  - 配置 30 秒硬性超时，超时后中断请求并返回超时错误码
  - Observable：合法链接在 30 秒内返回 HTML 字符串；超时场景返回 FETCH_TIMEOUT 错误
  - _Boundary: XianyuScraper_
  - _Requirements: 2.4, 2.7, 6.1_

- [x] 2.3 (P) 实现商品信息 HTML 解析与字段提取
  - 从 HTML 中提取商品标题、价格、描述文本、主图 URL 列表
  - 若页面包含成色文案，按关键词映射到平台四项标准成色之一
  - 价格文本清洗：去除货币符号与分隔符，解析为数值
  - Observable：给定包含商品信息的 HTML 片段，返回结构化的商品数据对象
  - _Boundary: XianyuScraper_
  - _Requirements: 2.5, 2.6, 2.9_

- [x] 2.4 组装抓取服务：编排校验、抓取、解析与错误处理
  - 将链接校验、HTTP 抓取、HTML 解析串联为完整抓取流程
  - 任一环节失败时立即终止并返回对应标准化错误信息
  - 对未知异常做安全捕获，降级为解析失败错误
  - Observable：输入任意闲鱼链接，都能在 5 秒内得到明确的抓取成功或标准化失败结果
  - _Depends: 2.1, 2.2, 2.3_
  - _Boundary: XianyuScraper_
  - _Requirements: 2.4, 2.7, 2.8, 2.9, 6.1_

## 3. Core：数据持久化与 API 入口

- [x] 3.1 (P) 实现基于闲鱼链接的商品重复性检查
  - 查询数据库中是否已存在相同闲鱼链接且未删除的商品记录
  - Observable：对已存在的闲鱼链接返回"已存在"标志；对新链接返回"可创建"
  - _Boundary: ProductRepository_
  - _Requirements: 1.3, 6.2_

- [x] 3.2 实现抓取入口：编排验证、去重、抓取并返回标准化响应
  - 接收用户提交的闲鱼链接，依次执行格式校验、重复性检查、商品抓取
  - 返回明确的 HTTP 状态：成功(200)、格式无效(400)、已存在(409)、解析失败(422)、超时(504)
  - Observable：通过 API 测试工具调用入口，每种场景均返回与状态码匹配的结构化响应
  - _Depends: 2.4, 3.1_
  - _Boundary: ScrapeAPI Route_
  - _Requirements: 1.1, 1.2, 1.3, 2.4, 6.1_

## 4. Core：前端交互实现

- [x] 4.1 实现 sell 页面的闲鱼链接输入、搬运触发与加载态
  - 在现有发布表单上方新增闲鱼链接输入区域与"搬运"操作按钮
  - 点击搬运后禁用按钮并展示加载指示器，防止重复提交
  - Observable：用户粘贴链接并点击搬运，按钮立即进入禁用加载状态
  - _Boundary: SellPage_
  - _Requirements: 1.1, 3.1_

- [x] 4.2 实现抓取结果到发布表单的数据预填与用户编辑保留
  - 抓取成功后，将标题、价格、描述、图片链接、成色自动填充到对应表单字段
  - 预填后用户仍可自由修改任意字段
  - 图片区域展示抓取到的预览图；若图片加载失败则展示占位图
  - Observable：抓取成功后表单各字段被正确填充；用户修改字段后发布，数据以修改后为准
  - _Depends: 4.1_
  - _Boundary: SellPage_
  - _Requirements: 3.1, 3.2, 3.3, 4.2, 4.3_

- [x] 4.3 实现抓取异常与重复链接的用户提示与引导
  - 格式无效时提示"请检查链接格式"
  - 链接已存在时提示"该商品已搬运"并展示查看已有商品的入口
  - 抓取超时/失败/解析失败时提示对应的友好文案，引导用户手动填写
  - Observable：每种异常场景下，用户都能在输入区域下方看到清晰的中文提示与下一步引导
  - _Depends: 4.1_
  - _Boundary: SellPage_
  - _Requirements: 1.2, 1.3, 2.7, 2.8, 2.9, 3.3_

## 5. Integration：发布流程打通与联调

- [x] 5.1 扩展商品发布接口以接收来源标记与原始链接
  - 扩展现有商品发布接口，接收并持久化 `source`（固定为"xianyu"）和 `xianyuUrl`
  - 发布前校验价格大于 0 且至少包含 1 张图片
  - Observable：基于搬运数据提交发布后，数据库中该商品记录包含正确的来源标记和原始链接
  - _Boundary: ProductRepository, PublishAPI_
  - _Requirements: 4.1, 4.2, 4.3, 6.2_

- [x] 5.2 联调 sell 页面与抓取 API 的端到端数据流
  - 从粘贴链接、点击搬运、预填表单、确认发布到数据持久化的完整链路走通
  - 验证发布后的商品在商品列表中正常展示，且来源信息正确
  - Observable：在本地完整执行一次"粘贴闲鱼链接 → 搬运 → 确认发布 → 查看列表"全流程无阻断
  - _Depends: 3.2, 4.2, 4.3, 5.1_
  - _Boundary: Integration_
  - _Requirements: 1.1, 3.1, 4.1, 4.2, 4.3_

- [x] 6.1 验证合法闲鱼链接的完整抓取与预填流程
  - 使用真实闲鱼商品链接测试，验证标题、价格、图片、描述被正确抓取并预填
  - 验证发布后的商品在 marketplace 页面可见且数据完整

  - _Boundary: Validation_
  - _Requirements: 2.4, 2.5, 3.1, 4.1_

- [x] 6.2 验证非法链接、重复链接、抓取超时等异常场景的错误提示
  - 输入非闲鱼链接，验证前端提示"请检查链接格式"
  - 重复搬运同一链接，验证前端提示"该商品已搬运"并提供跳转
  - 模拟抓取超时/失败，验证前端展示对应降级文案
  - Observable：每种异常场景均有截图或日志证明用户看到了正确的提示信息
  - _Boundary: Validation_
  - _Requirements: 1.2, 1.3, 2.7, 2.8, 2.9, 3.3_
