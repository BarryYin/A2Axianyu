# 需求文档: 平台代购管理员页面

## 1. 管理员面板需求

### REQ-001: 待办订单队列
当管理员访问面板时，系统应展示按创建时间倒序排列的所有待办订单，显示：订单ID、商品标题、闲鱼链接、成交价、买家信息、订单创建时间。

### REQ-002: 一键跳转闲鱼
当管理员点击订单上的闲鱼链接按钮时，系统应在新标签页打开闲鱼原始商品页面。

### REQ-003: 订单状态筛选
当管理员选择状态筛选器（待采购/已采购/已发货/已送达/失败/已退款）时，系统应更新面板仅显示匹配状态的订单。

### REQ-004: 订单详情查看
当管理员点击某条订单时，系统应展示详情：完整商品信息、议价历史、买家收货地址、支付状态、操作按钮。

### REQ-005: 面板统计数据
当面板加载时，系统应显示汇总统计：今日待办数、今日已处理数、成功率、平均处理时间。

## 2. 订单管理需求

### REQ-006: 确认采购
当管理员确认闲鱼采购成功时，系统应将订单状态从"待采购"变更为"已采购"，并要求填写：快递单号、物流公司、可选备注。

### REQ-007: 记录采购失败
当管理员标记订单为失败时，系统应要求选择失败原因（涨价/已售出/卖家不回复/商品问题/其他）和可选备注，然后将订单变更为"失败"状态。

### REQ-008: 状态流转验证
当管理员尝试变更订单状态时，系统应验证状态流转是否合法：待采购→已采购、待采购→失败、已采购→已发货、已发货→已送达、失败→已退款。

### REQ-009: 物流信息更新
当管理员更新已采购订单的物流信息时，系统应验证快递单号格式并更新订单记录，附带时间戳。

### REQ-010: 批量处理
当管理员选中多条待办订单时，系统应允许批量状态更新（批量标记已采购或失败），单次确认操作。

## 3. 买家通知需求

### REQ-011: 采购成功通知
当订单标记为"已采购"时，系统应自动通过买家的AI代理发送通知，包含：订单确认、快递单号、物流公司、预计到货时间、商品摘要。

### REQ-012: 采购失败通知+退款
当订单标记为"失败"时，系统应自动通知买家AI代理：失败原因、退款确认、退款金额、预计退款时间。

### REQ-013: 推荐类似商品
当采购失败时，系统应根据原商品的品类、价格区间、关键词查询3-5个类似商品，将推荐包含在失败通知中。

### REQ-014: 物流状态更新
当订单状态变更为"已发货"或"已送达"时，系统应通知买家AI代理最新状态和物流信息。

### REQ-015: 通知送达追踪
当买家通知发送后，系统应记录通知送达状态（已发送/已送达/已读）和时间戳。

## 4. 商品上架需求

### REQ-016: 手动提交闲鱼链接
当用户提交闲鱼商品URL时，系统应验证URL格式，提取商品元数据（标题、价格、图片、卖家信息），创建商品列表并设置AI议价价格区间。

### REQ-017: 平台Agent自动采集
当平台Agent扫描闲鱼匹配用户兴趣的商品时，系统应自动创建商品列表，配置提取的元数据和可配置的议价价格区间。

### REQ-018: 商品列表验证
当新商品上架时，系统应验证：闲鱼URL有效、非重复上架（按URL去重）、至少1张图片、价格大于0。

## 5. AI议价需求

### REQ-019: 议价价格区间配置
当商品上架时，系统应允许配置：最低可接受价格、最高可接受价格、自动接受阈值（AI代理无需人工介入直接接受的价格）。

### REQ-020: Agent间议价规则
当买家AI代理发起议价时，系统应执行：最多5轮议价、每轮价格调整幅度5-10%、单次议价超时10分钟。

### REQ-021: 自动接受阈值
当议价价格落入自动接受区间（可按商品配置，默认为标价的90%以上）时，系统应自动接受成交并生成待办订单。

### REQ-022: 议价历史记录
当AI议价发生时，系统应记录所有议价轮次：时间戳、出价金额、还价金额、代理响应、最终结果。

## 6. API端点需求

### REQ-023: 订单管理API
系统应提供RESTful API端点：
- GET /api/admin/orders — 分页+筛选查询订单
- GET /api/admin/orders/:id — 获取订单详情
- PATCH /api/admin/orders/:id/status — 更新订单状态
- POST /api/admin/orders/:id/tracking — 添加物流信息
- POST /api/admin/orders/bulk-update — 批量状态更新

### REQ-024: 商品管理API
系统应提供RESTful API端点：
- POST /api/products — 提交新商品链接
- GET /api/products — 分页+筛选查询商品
- GET /api/products/:id — 获取商品详情
- PATCH /api/products/:id — 更新商品配置

### REQ-025: 通知API
系统应提供RESTful API端点：
- POST /api/notifications/send — 向买家AI代理发送通知
- GET /api/notifications/:orderId — 获取订单通知历史
- PATCH /api/notifications/:id/status — 更新通知送达状态

### REQ-026: 议价API
系统应提供RESTful API端点：
- POST /api/negotiations/start — 发起议价会话
- GET /api/negotiations/:id — 获取议价状态和历史
- POST /api/negotiations/:id/offer — 提交还价
- POST /api/negotiations/:id/accept — 接受当前报价
