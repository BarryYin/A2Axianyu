# A2A闲鱼 — 商品供给与成交流程（已确认方案）

> 2026-04-18 | 状态：已确认

---

## 核心模式：AI 谈判 + 人工采购 + AI 通知

```
AI 谈判 → 买家付款 → 平台生成待办 → 人工去闲鱼下单 → 填回结果 → AI 通知买家
```

AI 管"谈"，人管"买"，AI 管"通知"。

---

## 完整流程

### 1. 供给端：商品信息获取

平台卖家 Agent 从闲鱼获取商品信息，上架到 A2A闲鱼。
- 用户粘贴闲鱼链接 → 自动解析上架
- 平台 Agent 定期扫描热门品类 → 批量上架
- 标注来源：闲鱼原始链接

### 2. 交易端：AI Agent 自动议价

```
买家 Agent 浏览商品
    ↓
AI Agent 自动议价（基于买卖双方设定的规则）
    ↓
在成交范围内 → 秒成交
不在范围内 → counter-offer 或拒绝
    ↓
买家确认 + 付款
```

### 3. 采购端：人工执行（Human-in-the-loop）

```
A2A 买家成交 + 付款
    ↓
平台生成待办订单：
  ├── 商品信息（名称、成交价、买家信息）
  ├── 闲鱼原始链接
  └── 状态：待采购
    ↓
管理员看到待办列表，点击闲鱼链接
    ↓
管理员手动在闲鱼完成下单
    ↓
管理员回来选择操作结果：
  ├── [已购] → 输入快递单号+物流公司
  └── [失败] → 选择原因（涨价/已售出/其他）
```

### 4. 交付端：AI 通知买家

```
已购：AI 通知买家"已采购，快递已发出"+ 物流信息
失败：自动退款 + 推荐类似商品 + 通知买家
```

---

## 为什么允许失败

- 闲鱼本身就是 C2C 平台，卖家不卖是常态
- 退款机制用户有预期（类似淘宝/闲鱼取消订单）
- 最坏情况就是退款，平台无资金损失
- 关键是提高成功率，不是追求 100%

---

## 关键指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 代购成功率 | > 60% | 去闲鱼下单能成交的比例 |
| 成交到发货时间 | < 2 小时 | 买家付款后多久能发货 |
| 退款率 | < 40% | 用户能接受，但要尽量低 |
| 退款后推荐转化 | > 30% | 退款时推荐类似商品，用户继续购买 |

## 技术要点

### 管理员页面（新增）
- 待办订单列表（按时间排序）
- 每个订单显示：商品信息 + 闲鱼原始链接 + 成交价
- 一键跳转闲鱼链接
- 操作按钮：已购（需填快递信息）/ 失败（需选原因）
- 状态流转：待采购 → 已购/失败 → 已通知买家

### 订单状态机
```
pending_purchase → purchased → shipped → delivered
                → failed → refunded
```

### 买家端通知
- 待采购：等待平台从闲鱼采购
- 已购：快递信息 + 物流跟踪
- 失败：已退款 + 推荐商品链接

---

## 不做的事（MVP 范围外）

- ❌ 浏览器自动化在闲鱼下单（合规风险）
- ❌ 自动锁定闲鱼卖家（复杂度高）
- ❌ 囤货模式（资金风险）
- ❌ 复杂的物流对接（人工处理即可）
